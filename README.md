## Features

- **Progressive Unlock** вҖ” Solve Easy problems to unlock Medium
- **Live Timer** вҖ” 20-minute server-authoritative countdown
- **Opponent Awareness** вҖ” See test cases passed, wrong submissions, current problem вҖ” never their code
- **ELO Matchmaking** вҖ” Classic Elo rating system with skill-based pairing
- **3 Languages** вҖ” C++, Python, Java via Judge0 sandbox
- **Real-time Battle** вҖ” Socket.IO-powered opponent progress feed with Redis adapter for horizontal scaling
- **Code Editor** вҖ” In-browser Monaco editor with syntax highlighting
- **400+ Real Problems** вҖ” Sourced from CSES Problem Set
- **ELO Rank Tiers** вҖ” Bronze вҶ’ Silver вҶ’ Gold вҶ’ Platinum вҶ’ Diamond вҶ’ Master вҶ’ Grandmaster
- **Win Streaks** вҖ” Track hot streaks with visual indicators
- **Practice Mode** вҖ” Play against AI bot with Easy/Medium difficulty
- **Google OAuth** вҖ” Sign in with Google or email/password
- **Anonymous Feedback** вҖ” Built-in feedback system
- **Interactive Terminal Landing** вҖ” Type commands, explore the platform
- **Music Player** вҖ” Playlist with loop-all, mute toggle, persisted preferences

## Architecture

```
Browser в”Җв”ҖHTTPS/WSSв”Җв”Җ> Nginx в”Җв”Җв–ә Next.js (:3000)
                                в””в”ҖRESTв”Җв”Җ> PostgreSQL
                                в””в”ҖRESTв”Җв”Җ> Redis (queue, limits, adapter)
                                в””в”Җenqueueв”Җв”Җ> matchmaker в”Җв”Җв–ә createMatch
                                в””в”ҖRedis pub/subв”Җв”Җ> Socket.IO (any instance)

Next.js (:3000) в”Җв”ҖRESTв”Җв”Җ> Judge0 (:2358) в”Җв”Җ> isolate sandbox

Workers (separate processes):
  - realtime   (Socket.IO server on :3002, Redis adapter)
  - matchmaker (Redis sorted set, HMAC-signed emit bridge)
  - finalizer  (SELECT FOR UPDATE, idempotent, emits match:end)
  - bot        (practice AI, recursive setTimeout, graceful shutdown)
```

### Monorepo (pnpm workspaces)

| Path | Package | Purpose |
|------|---------|---------|
| `apps/web` | `web` | Next.js app (frontend + REST API) + Socket.IO server + workers |
| `packages/db` | `@zapdos/db` | Prisma schema, client singleton, migrations |
| `packages/elo` | `@zapdos/elo` | Pure Elo rating math |
| `packages/realtime` | `@zapdos/realtime` | Shared Socket.IO event contracts |
| `packages/judge` | `@zapdos/judge` | Judge0 client, token-based output compare, verdict logic |
| `packages/match` | `@zapdos/match` | Matchmaking queue, scoring, match lifecycle |

## Getting Started (Local Development)

```bash
# 1. Clone and install
git clone https://github.com/Vedang-P/cp-battle.git
cd zapdos
pnpm install

# 2. Copy env and fill in secrets
cp .env.example .env
# Edit .env вҖ” set AUTH_SECRET and NEXTAUTH_SECRET to random strings

# 3. Start infrastructure (postgres, redis, Judge0)
pnpm infra:up
# Judge0's first run may take ~30s to be ready

# 4. Apply DB schema + seed problems
pnpm db:generate
pnpm db:push

# 5. Scrape 400+ real problems from CSES
pnpm scrape-problems -- --cses

# 6. Start all services (each in a separate terminal, or use start-dev.sh)
pnpm dev                # Next.js on :3000
pnpm dev:realtime       # Socket.IO on :3002
pnpm dev:matchmaker     # Matchmaking worker
pnpm dev:finalizer      # Match finalization worker
pnpm dev:bot            # Practice bot worker
```

Or start all at once:
```bash
bash scripts/start-dev.sh
```

Open [http://localhost:3000](http://localhost:3000).

Verify infrastructure at [http://localhost:3000/api/health](http://localhost:3000/api/health) вҖ” all three services should report `ok`.

## Match Rules

| Rule | Value |
|------|-------|
| **Format** | Progressive unlock вҖ” Easy вҶ’ Medium |

## Problem Corpus

Zapdos uses **real problems** sourced from the CSES Problem Set вҖ” not generated.

| Source | Count | How to add |
|--------|-------|------------|
| CSES Problem Set | 394 | `pnpm scrape-problems -- --cses` |
| **Total** | **394+** | |

### Scraping problems

```bash
pnpm scrape-problems -- --cses      # CSES (394 problems)
pnpm scrape-problems -- --atcoder   # AtCoder ABC (1000+ problems)
pnpm scrape-problems                # all sources
```

The scraper:
- Fetches real problem statements + sample test cases
- Saves to DB in batches of 10 (progress is never lost)
- Live progress output with timestamps
- Upserts by slug (re-running updates existing problems)
- Biases toward EASY/MEDIUM problems

### Adding your own music

Drop MP3 files into `apps/web/public/audio/music/` and update `apps/web/public/audio/manifest.json`:

```json
{
  "tracks": [
    { "file": "my-track-1.mp3", "title": "My Track 1" },
    { "file": "my-track-2.mp3", "title": "My Track 2" }
  ]
}
```

The music player loops through all tracks and back to the start. Users can mute/unmute from the navbar.

## Security

- **HMAC-signed /emit bridge** вҖ” all internal Socket.IO event emissions are signed with AUTH_SECRET
- **JWT-expiry verification** вҖ” Socket.IO tokens expire after 1 hour
- **CSRF protection** вҖ” Origin/Referer check on all mutating API routes
- **Row-level locking** вҖ” `SELECT FOR UPDATE` on Match and User rows during finalization
- **Atomic rate limiting** вҖ” Lua scripts for INCR+PEXPIRE (no orphaned keys)
- **Sanitized opponent data** вҖ” compile/runtime errors never sent to the opponent
- **Judge0 sandbox** вҖ” `enable_network: false`, per-process time/memory limits
- **bcrypt password hashing** вҖ” cost factor 12
- **Disposable email blocklist** вҖ” 200+ domains blocked at signup
- **Input validation** вҖ” Zod schemas on all API inputs
